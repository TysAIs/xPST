# XPST Enterprise Ship-Readiness Audit — 2026-06-10

Branch `feat/knowledge-base` (PR #4 to `main`). Audited on Linux (DGX Spark, Python 3.13 venv). This report replaces the audit run that crashed on 2026-06-10 before writing its output.

## Method

Six deterministic gates run to completion, then seven parallel read-only audit lanes (tests, security, cross-platform, enterprise, opensource, kb, quality), each scored 0-10 with severity-tagged findings, then every CRIT/HIGH re-verified against ground truth in the source to remove false positives. Lanes were instructed that unsupported high-severity findings would be discarded, and several were.

## Top-10 Prioritized Fix List

Priority order. CRIT count is zero. Items 1-7 are HIGH, 8-10 are MED that are either trivial or explicit ship follow-ups.

1. **[HIGH] Windows unicode caption crash.** `schedule_manager.py:73,88` open the schedule file without `encoding="utf-8"` while `_save` serializes with `ensure_ascii=False`. Any emoji or CJK caption raises `UnicodeEncodeError` on Windows. Fix: add `encoding="utf-8"` to both opens. STATUS: done (commit c2f8fd8 — both schedule_manager.py opens now pass encoding="utf-8"; tests/test_schedule_unicode.py covers the emoji/CJK round-trip).
2. **[HIGH] Dashboard unsafe default.** `dashboard/server.py:181` defaults `host="0.0.0.0"` with auth only when credentials are set, and the `dashboard` CLI command has no `--host` flag though README documents one. Fix: default to `127.0.0.1`, add `--host`, warn when binding non-loopback without auth. STATUS: done (start_dashboard default now 127.0.0.1, non-loopback bind without auth logs a WARNING, dashboard CLI gained --host, docs reconciled, tests/test_dashboard.py added).
3. **[HIGH] KB ghost-area accumulation.** `models.py:96` derives `Area.id` from the LLM-generated label and `json_store.upsert_area` keys by id, so a re-phrased label spawns a new area every organize run. Reproduced. Fix: key `Area.id` on sorted nugget membership, treat label as mutable display. STATUS: done (commit b2a15e0 — Area.id now keyed on sorted nugget membership with empty-membership label fallback, organize_store reconciles/prunes stale areas and clears dangling area_ids; regression tests in tests/test_knowledge_models.py, tests/test_knowledge_organize.py, tests/test_knowledge_store_contract.py).
4. **[HIGH] Ragged centroid silent truncation.** `organize/_vectors.py:centroid` truncates ragged vectors to the shortest member and divides by full count, producing a corrupt centroid and never raising. Reproduced. Fix: validate equal dimensionality and raise on mismatch. STATUS: done (commit b2a15e0 — organize/_vectors.py:centroid now raises ValueError on mixed widths instead of truncating; regression coverage in tests/test_knowledge_vectors.py).
5. **[HIGH] Dead duplicate MCP server ships.** `src/xpst/mcp_server.py` (359 lines) is imported by nothing. The live entrypoint is `xpst.mcp.server`. Fix: delete the dead module. STATUS: done (commit 2560d55 — deleted the dead duplicate `src/xpst/mcp_server.py`; the file no longer exists and nothing imports it).
6. **[HIGH] Divergent CrossPostEngine.** CLI, scheduler, desktop, and package init import `xpst.engine`, while `mcp/server.py` imports `xpst.engine_v2`. Behavior can diverge across surfaces. Fix: consolidate onto one engine and retire the other. STATUS: done (retired `xpst.engine_v2`, migrated `mcp/server.py` to the canonical `xpst.engine.CrossPostEngine` with behavior-equivalent handlers, deleted the v2 module, reconciled docs/api.md + docs/README.md, added tests/test_engine_consolidation.py asserting a single engine source of truth across CLI/scheduler/desktop/MCP/package-init).
7. **[HIGH] LGPL desktop-bundle compliance.** `release_artifacts.py:copy_project_documents` ships LICENSE, NOTICES, LICENSING_REPORT but no Qt/PySide6 LGPL text or relink offer, contradicting the project's own LICENSING_REPORT guidance. Fix: add a Qt LGPL notice, bundle it, gate it in `verify_desktop_package.py`. STATUS: done (added NOTICES_QT_LGPL.md with LGPL-3.0 attribution + written relink offer per LICENSING_REPORT, made it a required bundled document in copy_project_documents and tracked it in release evidence, added a verify_desktop_package gate so a desktop package missing/incomplete on the LGPL notice fails, added tests in tests/test_release_artifacts.py and tests/test_repo_assets.py).
8. **[MED] xpst build wrong spec on Linux.** `cli.py:2738` maps `linux` to `build_macos.spec`. Fix: map to `build_linux.spec`. STATUS: done (commit c2f8fd8 — linux now maps to build_linux.spec; tests/test_build_spec_map.py guards the mapping).
9. **[MED] W4 QML glyph rendering.** `AnalyticsPage.qml` concatenates a Lucide glyph and label into one Text without the icon font family, and `Icons.qml` carries a redundant `FontLoader` nothing references. Fix: split glyph and label Texts with `font.family: theme.iconFontFamily`, remove the redundant loader. STATUS: pending (not part of this run; partially addressed by commit 211f925 which made Icons.qml the font-family source of truth and put the FontLoader to use, but `AnalyticsPage.qml:205` still concatenates `modelData.icon + modelData.name` in one Text without an icon font family — those tab entries are plain text labels, so the visible defect is cosmetic, but the stated split was not applied. Left open and honest.).
10. **[MED] Plaintext YouTube token + credential-path coverage.** `sessions.py:111` and `youtube.py:131` write the OAuth token in cleartext at default permissions, contradicting SECURITY.md. The credential-writing onboarding paths (`connect.py` at 0%, CLI `auth`) have no tests. Fix: harden the token file to 0600 and align the doc, add boundary-mocked tests for connect and auth. STATUS: done (added utils/secure_io.write_text_0600 and routed every token/session/cookie write through it in sessions.py + youtube.py + connect.py; twikit cookies chmod'd post-login; SECURITY.md now documents the 0600 compatibility files; fixed a latent unbound-`json` bug in connect_instagram; deleted the dead, zero-importer xpst.auth.auth_manager module + package and dropped its orphaned authlib dependency; added tests/test_token_hardening.py with boundary-mocked connect/auth/session coverage asserting 0600).
