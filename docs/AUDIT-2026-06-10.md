## Method

Lanes were instructed that unsupported high-severity findings would be discarded, and several were.

## Top-10 Prioritized Fix List

Priority order. CRIT count is zero. Items 1-7 are HIGH, 8-10 are MED that are either trivial or explicit ship follow-ups.

1. **[HIGH] Windows unicode caption crash.** `schedule_manager.py:73,88` open the schedule file without `encoding="utf-8"` while `_save` serializes with `ensure_ascii=False`. Any emoji or CJK caption raises `UnicodeEncodeError` on Windows. Fix: add `encoding="utf-8"` to both opens. STATUS: pending.
2. **[HIGH] Dashboard unsafe default.** `dashboard/server.py:181` defaults `host="0.0.0.0"` with auth only when credentials are set, and the `dashboard` CLI command has no `--host` flag though README documents one. Fix: default to `127.0.0.1`, add `--host`, warn when binding non-loopback without auth. STATUS: done (start_dashboard default now 127.0.0.1, non-loopback bind without auth logs a WARNING, dashboard CLI gained --host, docs reconciled, tests/test_dashboard.py added).
3. **[HIGH] KB ghost-area accumulation.** `models.py:96` derives `Area.id` from the LLM-generated label and `json_store.upsert_area` keys by id, so a re-phrased label spawns a new area every organize run. Reproduced. Fix: key `Area.id` on sorted nugget membership, treat label as mutable display. STATUS: pending.
4. **[HIGH] Ragged centroid silent truncation.** `organize/_vectors.py:centroid` truncates ragged vectors to the shortest member and divides by full count, producing a corrupt centroid and never raising. Reproduced. Fix: validate equal dimensionality and raise on mismatch. STATUS: pending.
5. **[HIGH] Dead duplicate MCP server ships.** `src/xpst/mcp_server.py` (359 lines) is imported by nothing. The live entrypoint is `xpst.mcp.server`. Fix: delete the dead module. STATUS: pending.
6. **[HIGH] Divergent CrossPostEngine.** CLI, scheduler, desktop, and package init import `xpst.engine`, while `mcp/server.py` imports `xpst.engine_v2`. Behavior can diverge across surfaces. Fix: consolidate onto one engine and retire the other. STATUS: pending.
7. **[HIGH] LGPL desktop-bundle compliance.** `release_artifacts.py:copy_project_documents` ships LICENSE, NOTICES, LICENSING_REPORT but no Qt/PySide6 LGPL text or relink offer, contradicting the project's own LICENSING_REPORT guidance. Fix: add a Qt LGPL notice, bundle it, gate it in `verify_desktop_package.py`. STATUS: pending.
8. **[MED] xpst build wrong spec on Linux.** `cli.py:2738` maps `linux` to `build_macos.spec`. Fix: map to `build_linux.spec`. STATUS: pending.
9. **[MED] W4 QML glyph rendering.** `AnalyticsPage.qml` concatenates a Lucide glyph and label into one Text without the icon font family, and `Icons.qml` carries a redundant `FontLoader` nothing references. Fix: split glyph and label Texts with `font.family: theme.iconFontFamily`, remove the redundant loader. STATUS: pending.
10. **[MED] Plaintext YouTube token + credential-path coverage.** `sessions.py:111` and `youtube.py:131` write the OAuth token in cleartext at default permissions, contradicting SECURITY.md. The credential-writing onboarding paths (`connect.py` at 0%, CLI `auth`) have no tests. Fix: harden the token file to 0600 and align the doc, add boundary-mocked tests for connect and auth. STATUS: pending.
